Bugs críticos
BUG 1
Batch ingest se detiene en el primer error (es tu bug reportado)
El try/except envuelve todo el for loop. Cualquier excepción (timeout de LM Studio, JSON inválido del modelo, encoding error) mata el batch completo. Si el primer archivo falla, los demás nunca se procesan.
try: for idx, file_path in enumerate(file_paths): # ← todo el loop adentro ...process... except Exception as exc: self.after(0, lambda: self._set_status(f"❌ Batch detenido en {processed}..."))
Fix: mover el try/except DENTRO del for loop, con continue en el except para seguir con el siguiente archivo.
BUG 2
Obsidian graph en estrella: las páginas no tienen wikilinks entre sí
En toda la wiki hay exactamente 1 wikilink [[...]] (en el archivo de demo de Obsidian). Las páginas generadas por el modelo usan texto plano como `wiki/fuentes/file.md` en bloques de código, no links reales. Obsidian solo puede trazar conexiones con [[wikilinks]] o links markdown. El único archivo con links reales es index.md, de ahí la topología de estrella.
Trazabilidad - Fuente original: `wiki/fuentes/hype-usdc-chart.md` ← texto plano, Obsidian lo ignora Debería ser: - Fuente original: [[hype-usdc-chart-2026-04-09]] ← esto sí construye la conexión
Fix: agregar al schema del prompt una regla explícita: "Usá siempre wikilinks Obsidian [[nombre-archivo]] para referencias cruzadas entre páginas, nunca texto plano ni código."
Bugs menores
BUG 3
parse_json_response: rfind("\n{") puede cortar el JSON a la mitad
Busca el último \n{ del string. Si el JSON tiene objetos anidados en nuevas líneas, apunta a un objeto interno en lugar del objeto raíz, produciendo JSON inválido. Caso típico: cuando el contenido markdown de una página empieza con { al inicio de una línea.
start = raw.rfind("\n{") # ← apunta al ÚLTIMO \n{ del JSON, puede ser un objeto anidado end = raw.rfind("}") + 1 # ← también corta en el último } del string
Fix: usar raw.find("{") (primero) y buscar el } final con un contador de balance de llaves, o usar json.JSONDecoder().raw_decode().
BUG 4
_HOY / _HORA calculados al inicio, nunca se actualizan
Son constantes de módulo. Si la app corre más de unas horas (o cruza la medianoche), todos los ingest y consultas del día siguiente se marcan con la fecha de arranque.
_HOY = datetime.now().strftime("%Y-%m-%d") # ← se fija al importar el módulo
Fix: calcular dentro de load_schema() con datetime.now() cada vez que se llama.
BUG 5
normalize_route: path traversal con str.startswith()
Un path como /ruta/al/wiki-malicioso/archivo.md pasaría el chequeo porque la comparación es string, no path. Debería usarse Path.is_relative_to().
if not (str(resolved).startswith(str(WIKI_DIR)) or ...): # ← bug de string vs path raise ValueError(...) # wiki-extra pasa el check de wiki incorrectamente: # "/path/wiki-extra".startswith("/path/wiki") → True
Fix: resolved.is_relative_to(WIKI_DIR) or resolved.is_relative_to(RAW_DIR) (Python 3.9+).
BUG 6
Batch ingest re-archiva archivos que ya estaban en raw/
Al usar "Cargar raw/", los archivos ya están archivados. Pero process_ingest_source llama a archive_source() nuevamente, creando duplicados con timestamp en el nombre. Además, "Cargar raw/" solo carga de raw/fuentes-originales/, ignorando PDFs e imágenes en raw/assets/.
Fix: agregar un flag already_archived=False a process_ingest_source. Al cargar desde raw/, pasarlo como True.
Problemas de UI / UX
UI 1
Batch ingest: no hay progreso real ni posibilidad de cancelar
El status bar muestra "Batch ingest 2/5: archivo.md" pero si LM Studio tarda 3 minutos por archivo y hay 8 archivos, la app parece colgada. No hay barra de progreso visual ni botón de cancelar. El thread es daemon=True, así que matar la ventana cancela todo sin warning.
UI 2
El editor es solo lectura en la práctica, pero parece editable
El ScrolledText está en state="normal" y el usuario puede tipear en él, pero esos cambios nunca se guardan (no hay botón de "guardar archivo"). Genera expectativa falsa. Debería ser state="disabled" con un botón explícito de edición, o tener auto-save real.
Mejoras sugeridas
MEJ 1
El schema debería incluir entidades y conceptos ya existentes como contexto
Al ingerir, el modelo no sabe qué entidades ya existen en wiki/entidades/ o wiki/conceptos/. Debería recibir el listado para reusar páginas en lugar de crear duplicados con nombres distintos.
MEJ 2
Agregar deduplicación de señales por similitud de título
Al ingerir varias fuentes sobre el mismo tema, el modelo genera múltiples señales casi idénticas. Una revisión simple de slugs o títulos antes de guardar evitaría la proliferación.
MEJ 3
Separar el timeout de ingest por tipo de archivo
Actualmente todos los ingest usan timeout=180. Un PDF de 50 páginas necesita más tiempo. Una imagen simple necesita menos. Sería útil un timeout configurable en la UI o calculado por tamaño de archivo.
MEJ 4
El lint debería poder archivar su propio reporte
El resultado del lint se muestra en el editor pero no se puede guardar fácilmente. Un botón "Guardar reporte de lint" en wiki/consultas/lint-YYYY-MM-DD.md sería útil para tracking evolutivo de la salud de la wiki.
MEJ 5
Soporte para drag & drop de archivos en la ventana de ingest
Tkinter soporta DnD via tkinterdnd2. Arrastrar un PDF o imagen directamente a la ventana de ingest mejoraría mucho el flujo de trabajo vs el file picker.
